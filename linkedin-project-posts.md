# LinkedIn Project Build Posts

## Post 1: AI Support Dashboard

I built and deployed an AI Support Dashboard to explore how AI can support real operational workflows.

The project focuses on support tickets, role-based views, SLA tracking, deployment monitoring, and secure AI-assisted response drafting.

What I learned:
- Keep API keys server-side, never in the browser
- Design dashboards around the decisions users need to make
- Deployment monitoring matters even for portfolio projects
- Role-based interfaces make the product feel more production-minded

Stack: Next.js, Supabase, Vercel, Tailwind CSS, OpenAI SDKs, GitHub

Live: https://ai-support-dashboard-navy.vercel.app
GitHub: https://github.com/obone410/AI-Support-Dashboard

## Post 2: PromptDeck AI

I built PromptDeck AI as a prompt management and AI workflow platform.

The goal was to move prompt engineering from scattered notes into a more structured system for reusable prompts, workflow organization, and practical AI operations.

What I learned:
- Prompt systems need structure, not just good wording
- Reusable prompt libraries make AI workflows easier to test
- Clear categories help users move faster
- Good UI matters when managing technical knowledge

Live: https://ai-prompt-management-platform.vercel.app
GitHub: https://github.com/obone410/AI-Prompt-Management-Platform

## Post 3: ResearchOS

I built ResearchOS to explore AI-assisted technical research workflows.

The project focuses on organizing research tasks, summarizing findings, and creating a more repeatable workflow for technical investigation.

What I learned:
- Research tools should reduce context switching
- AI output needs review, structure, and traceability
- A good research workflow is part product design, part systems design
- Technical research improves when notes, sources, and actions are connected

Live: https://ai-research-assistant-liart.vercel.app
GitHub: https://github.com/obone410/AI-Research-Assistant

## Post 4: Aegis-Monitor

I built Aegis-Monitor as a DevOps monitoring and deployment observability dashboard.

The project helped me practice thinking about system health, deployment status, incidents, and operational visibility from a product perspective.

What I learned:
- Monitoring dashboards should prioritize signal over decoration
- Deployment status needs to be easy to scan
- Incident-style layouts help communicate urgency
- DevOps fundamentals become clearer when you build tools around them

Live: https://devops-monitoring-dashboard-psi.vercel.app
GitHub: https://github.com/obone410/Aegis-Monitor
